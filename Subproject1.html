GUI for Business Operations - Python Project


Unfortunately I am not allowed to share the source code and the details of this project due to the agreement between me and my Employer.
Therefore I can only give you a brief overview below.


Program Focus and Main Objectives

The program specially developed for data tracking, analysis, reporting and visualization. The aim of this program is to make the daily item tracking activity much straightforward than in Excel and to completely separate the operative tracking activity from advanced reporting tasks. Somebody in a team might need just limited tracking functions and this can be done perfectly with this GUI. In the other hand, management can report anything from the Excel file but there are some basic built-in pivots, metrics and graphs in the program as well to support efficient data analysis. This can be used in sales, purchasing, finance tracking and in several other business areas.

Details

It is a GUI for the team members with limited functions to access the data where they can:
  • refresh the lists of items to edit so that they are aware of the daily workload in a structured way
  • edit/add data to the tracker by updating input fields

Every calculation needed for specific fields (target date, SLA computation, etc.) are running in the backend in a Pandas dataframe which loads the results into the Excel file.
Another part of the GUI makes reporting and analytics possible with some built-in pivots and graphs created by Matplotlib.
