GUI for Monthly Quality Assurance - Python Project

Unfortunately I am not allowed to share the source code and the details of this project due to the agreement between me and my Employer.
Therefore I can only give you a brief overview below.


Program Focus and Main Objectives

Quality Assurance is a process which can be implemented on a regular basis within business operational teams. Main purpose of this activity is to have an overview on the processing quality of the team against the standards.

Details

Target Audience: Managers, process leaders in business operations.

The team uses different Excel files for daily item tracking purposes. These items needs to be cross-checked - the team members check each other's completed items. The files need to be merged, filtered to relevant data, then random items need to be chosen based on different, multiple conditions in order to have a sample from each process/business area. Then the team checks the data and a report should be run, summarizing tables and graphs need to be created.
The program lets the user to browse for Excel tracker files (max 3) and to define the period.
The following operations can be performed afterwards:
  • merging of different Excel files into one dataframe
  • filtering the dataframe into relevant fields (e.g. completed items for a given period)
  • randomly choosing items for cross-checking based on multiple conditions
  • writing the random items into a new Excel file which the team can edit

Once the team edited the file and checked the data, the program can finish the work as following:
  • the result file can be browsed and selected
  • by hitting "Show Results" button, the Quality Assurance data are being analyzed
  • it creates some pivot tables in order to get an insight of the performance
  • it also generates graphs using matplotlib to visualize the team's processign quality for the given period
